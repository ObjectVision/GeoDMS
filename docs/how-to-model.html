Modelling in GeoDMS terms is the process of writing/editing a (set of) configuration file(s). In these files usually data-source is read and expression are configured.

The geodms-gui can be used as a tool in developing your model.

It is a good habit to also document your configuration.

project

A GeoDMS project usually consists of:

  • (a set of) configuration files
  • source data
  • documentation

configuration files

The GeoDMS uses a configuration language to configure source data, calculation steps, visualisation styles, export settings and relevant descriptive information. This information is stored in configuration-file.

declarative language

The GeoDMS configuration language is a declarative language. This means you configure what needs to be calculated and how the results needs to be presented. How calculations are performed in an efficient way is encapsulated in the implementation of each operators-and-functions that can be configured in an expression.
